# TFL 경기 관전 UI 1차 인터랙티브 프로토타입

이 폴더는 대표님 / 봉이사 검토를 위한 브라우저 확인용 인터랙티브 프로토타입이다.

```txt
목업 파일: docs/design/mockups/match-viewer-v1/index.html
스타일 파일: docs/design/mockups/match-viewer-v1/styles.css
샘플 데이터: docs/design/mockups/match-viewer-v1/sample-data.js
인터랙션 스크립트: docs/design/mockups/match-viewer-v1/prototype.js
```

## 목적

```txt
정적 목업만으로 판단하기 어려운 탭 전환, Play Focus View 움직임, 이벤트 연동 감각을 확인한다.
PC 가로 화면과 모바일 세로 화면에서 탭 기반 인게임 UI 사용감을 확인한다.
개발팀장 UI 구현 전 대표님 / 봉이사 컨펌 기준으로 사용한다.
```

## v0.4 보정 내용

```txt
공은 기본적으로 시작점에서 도착점까지 직선으로 이동한다.
숏패스 / 슛 / 골 슈팅은 꺾이지 않는 직선 경로로 표시한다.
롱패스 / 크로스도 x/y 경로는 직선에 가깝게 유지하고, 공중볼 느낌은 scale / shadow / timing으로만 표현한다.
공은 바둑알 중심과 겹치지 않고 항상 바둑알 외곽 발밑 또는 머리 바깥쪽에 위치한다.
바둑알끼리 겹치지 않도록 공격수 / 수비수 / 골키퍼의 최종 위치를 분산했다.
화면에 표시되는 모든 바둑알은 이벤트마다 작은 동시 움직임을 가진다.
패스 후보 바둑알은 같은 팀 색상을 유지하고 작은 라벨만 표시한다.
패스 / 크로스 / 롱패스 / 슛 도착점의 노란 타깃 링은 제거했다.
태클 샘플은 태클 성공으로 고정하고, 수비수 발밑으로 공 소유권이 넘어가는 결과를 명확히 표시한다.
크로스 샘플은 공격수 헤딩 연결로 고정해 누가 공을 처리했는지 보이게 했다.
골 이벤트는 슈팅 순간만 표시하며, 어시스트 선수는 pitch 위 active actor에서 제거하고 텍스트 정보로만 남겼다.
```

실시간 채팅, 커뮤니티, 서버 연동 구현 범위를 확정하는 의미가 아니다.

## 인게임 연출 기준

```txt
모바일 세로 화면에서도 그라운드는 세로형 경기장으로 만들지 않는다.
모바일은 세로 UI 안에 가로형 경기장을 카드처럼 배치한다.
선수 표현은 22명 전체 상시 표시 방식도 아니고 공 가진 선수 1명만 표시하는 방식도 아니다.
현재 플레이 이벤트에 관여한 선수만 표시하는 Play Focus View를 기준으로 한다.
```

Play Focus View 이벤트별 표시 기준:

```txt
드리블: 공 소유 선수 + 압박 수비수
태클: 공 소유 선수 + 태클 시도 선수
패스: 패스한 선수 + 패스 받을 선수
크로스: 크로스 올리는 선수 + 받을 선수 + 마크 수비수 + 골키퍼
롱패스: 패스한 선수 + 수신 후보 + 경합 수비수 + 골키퍼
슛: 슈팅 선수 + 골키퍼 + 필요 시 가까운 수비수
골: 득점 선수 + 골키퍼 + 필요 시 가까운 수비수
```

선수 강조 기준:

```txt
핵심 선수는 카드 UI로 크게 강조한다.
보조 선수는 바둑알, 작은 라벨, 등번호 중심으로 표시한다.
나머지 선수는 표시하지 않거나 흐릿한 배경 점 정도로만 처리한다.
목표는 FM식 22명 정밀 실시간 이동 구현이 아니라 TFL만의 이벤트 중심 플레이 포커스 연출로 보는맛을 만드는 것이다.
```

## 모바일 확인 방법

같은 파일을 브라우저에서 연 뒤 개발자도구 또는 반응형 보기에서 세로 화면으로 확인한다.

권장 확인 폭:

```txt
390px x 844px
430px x 932px
```

이 프로토타입은 HTML / CSS / JavaScript만 사용한다.
개발팀장이 그대로 복사해서 제품 코드로 사용하는 것이 아니라, 움직임과 탭 사용감 확인용 산출물이다.
